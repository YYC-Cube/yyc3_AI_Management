## 📋 概述

本文档详细说明 YanYu Cloud³ API Management 平台的环境变量配置，包括**本地开发环境**、测试环境和生产环境的设置要求。

## ⚡ 快速开始

### 1. 复制配置文件

```bash
cp .env.example .env.local
```

> ✅ **提示**: `.env.local` 已预置本地开发默认配置（PostgreSQL 5432端口, postgres用户, yyc3_management数据库）

### 2. 启动服务

```bash
# 后端 (端口 3001)
cd backend && npm run dev

# 前端 (端口 3000)
npm run dev
```

### 3. 验证配置

```bash
# 测试数据库连接
npm run env:test-db

# 测试 Redis 连接
npm run env:test-redis

# 完整环境检查
npm run env:test-all
```

### 🗄️ 数据库配置 (PostgreSQL 15)

**本地开发默认配置**：

| 变量名 | 描述 | 默认值 | 必需 |
|--------|------|--------|------|
| `DB_HOST` | 数据库主机 | `127.0.0.1` | ✅ |
| `DB_PORT` | 数据库端口 | `5432` | ✅ |
| `DB_NAME` | 数据库名称 | `yyc3_management` | ✅ |
| `DB_USER` | 数据库用户 | `yyc3_admin` | ✅ |
| `DB_PASSWORD` | 数据库密码 | （空） | ✅ |
| `DB_POOL_MAX` | 最大连接数 | `20` | ❌ |
| `DB_POOL_MIN` | 最小连接数 | `5` | ❌ |
| `DB_IDLE_TIMEOUT` | 空闲连接超时(ms) | `30000` | ❌ |
| `DB_CONNECT_TIMEOUT` | 连接超时(ms) | `10000` | ❌ |
| `DB_SSL` | 启用 SSL | `false` | ❌ |

#### 🔧 本地开发配置示例

```env
DB_HOST=127.0.0.1
DB_PORT=5432
DB_NAME=yyc3_management
DB_USER=yyc3_admin
DB_PASSWORD=your_secure_db_password_here
```

> ⚠️ **注意**:
> - 使用 `127.0.0.1` 而非 `localhost` 避免 IPv6 解析问题
> - 默认 PostgreSQL 端口为 **5432**（标准端口）
> - 用户名为 `yyc3_admin`（应用专用账户）
> - **本地开发必须设置密码**，不要留空
> - 连接池建议：开发环境 5-10，生产环境 20-50

#### 📊 连接池优化建议

```env
# 开发环境 (低负载)
DB_POOL_MAX=10
DB_POOL_MIN=2
DB_IDLE_TIMEOUT=30000

# 生产环境 (高并发)
DB_POOL_MAX=50
DB_POOL_MIN=10
DB_IDLE_TIMEOUT=60000
DB_CONNECT_TIMEOUT=10000
```

### 📦 Redis 缓存配置

缓存、会话存储和实时通信支持。

| 变量名 | 描述 | 默认值 | 必需 |
|--------|------|--------|------|
| `REDIS_HOST` | Redis 主机 | `localhost` | ✅ |
| `REDIS_PORT` | Redis 端口 | `6379` | ✅ |
| `REDIS_PASSWORD` | Redis 密码 | `yyc3_redis` | ❌ |
| `REDIS_DB` | 数据库索引 | `0` | ❌ |
| `REDIS_KEY_PREFIX` | 键前缀 | `yyc3:` | ❌ |
| `REDIS_TTL` | 默认过期时间(秒) | `3600` | ❌ |
| `REDIS_CLUSTER_MODE` | 集群模式 | `false` | ❌ |

#### 🔧 本地开发配置示例

```env
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=yyc3_redis
REDIS_DB=0
REDIS_KEY_PREFIX=yyc3:
REDIS_TTL=3600
REDIS_CLUSTER_MODE=false
```

> 💡 **提示**:
> - **键前缀** `yyc3:` 用于避免与其他应用冲突
> - **TTL 3600秒** (1小时) 适合会话缓存，可根据业务调整
> - **集群模式** 仅在生产环境高并发场景启用
> - 本地开发建议使用 Docker 启动 Redis：
>   ```bash
>   docker run -d --name yyc3-redis -p 6379:6379 redis:7-alpine --requirepass yyc3_redis
>   ```

### 🔐 认证与安全配置

JWT Token、密码加密、CORS 和限流等安全相关设置。

| 变量名 | 描述 | 示例值 | 必需 |
|--------|------|--------|------|
| `JWT_SECRET` | JWT 签名密钥 | `9e3416178f96...` | ✅ |
| `JWT_EXPIRES_IN` | Access Token 有效期 | `24h` | ❌ |
| `JWT_REFRESH_EXPIRES_IN` | Refresh Token 有效期 | `7d` | ❌ |
| `BCRYPT_ROUNDS` | bcrypt 加密轮次 | `12` | ❌ |
| `CORS_ORIGIN` | 允许的跨域源 | `http://localhost:3000` | ✅ |
| `RATE_LIMIT_WINDOW` | 限流窗口(分钟) | `15` | ❌ |
| `RATE_LIMIT_MAX` | 窗口内最大请求数 | `100` | ❌ |

#### 🔑 安全建议

```bash
# 生成安全的 JWT 密钥 (64字节十六进制)
openssl rand -hex 32
# 输出示例: 9e3416178f9677b1eadb7cc320b4d3e507f943d5be9c9f03d36eceedfad5dedf

# 生产环境必须使用强密钥
# ⚠️ 不要在代码中硬编码密钥！
# ⚠️ 不要在 .env.example 中填写真实密钥！
```

#### 🛡️ 限流策略说明

```env
# 开发环境 (宽松)
RATE_LIMIT_WINDOW=15          # 15分钟窗口
RATE_LIMIT_MAX=100            # 每15分钟最多100请求

# 生产环境 (严格)
RATE_LIMIT_WINDOW=1           # 1分钟窗口
RATE_LIMIT_MAX=60             # 每分钟最多60请求
```

### 🤖 OpenAI AI 配置

AI 功能相关配置（智能分析、对话等核心功能）。

| 变量名 | 描述 | 示例值 | 必需 |
|--------|------|--------|------|
| `OPENAI_API_KEY` | OpenAI API 密钥 | `sk-proj-...` | ✅ (AI功能) |
| `OPENAI_MODEL` | 默认模型 | `gpt-4o` | ❌ |
| `OPENAI_ORGANIZATION` | 组织ID | `org-xxx` | ❌ |
| `OPENAI_MAX_TOKENS` | 最大令牌数 | `4000` | ❌ |
| `OPENAI_TEMPERATURE` | 温度参数 (0-1) | `0.7` | ❌ |
| `OPENAI_TIMEOUT` | 请求超时(ms) | `30000` | ❌ |

#### 🔧 配置示例

```env
# 基础配置
OPENAI_API_KEY=sk-your-openai-api-key-here
OPENAI_MODEL=gpt-4o
OPENAI_ORGANIZATION=your-org-id-here

# 高级参数
OPENAI_MAX_TOKENS=4000        # 最大响应长度
OPENAI_TEMPERATURE=0.7         # 创意性 (0=确定性, 1=随机性)
OPENAI_TIMEOUT=30000           # 30秒超时
```

> 💡 **模型选择建议**:
> - **gpt-4o**: 最新多模态模型，推荐用于生产环境
> - **gpt-4-turbo**: 性价比高，适合批量处理
> - **gpt-3.5-turbo**: 成本最低，适合开发测试
> 
> ⚠️ **重要**: API Key 具有计费风险，请务必：
> - 设置**使用限额** (OpenAI Dashboard → Limits)
> - 监控**API 调用量**和费用
> - 定期**轮换密钥**

## ✅ 配置验证脚本

项目提供多个验证命令：

```bash
# 验证单个服务连接
```

## 🛡️ 安全最佳实践

### ✅ 推荐做法

1. **使用环境变量管理敏感信息**
   ```typescript
   // ✅ 正确
   const dbPassword = process.env.DB_PASSWORD;
   
   // ❌ 错误 - 永远不要硬编码
   const dbPassword = 'my-password';
   ```

2. **区分开发和生产配置**
   - 使用 `.env.local` 用于本地开发
   - 使用 `.env.production` 用于部署
   - 将 `.env*.local` 加入 `.gitignore`

3. **定期轮换密钥**
   - JWT 密钥每90天更换一次
   - 数据库密码每180天更换一次
   - API Key 泄露时立即更换

4. **最小权限原则**
   - 数据库用户仅授予必要权限
   - Redis 不使用 root 账户
   - API Key 限制 IP 和使用范围

### ❌ 常见错误

1. **将 `.env` 文件提交到版本控制**
2. **在客户端代码中使用服务端密钥**
3. **使用弱密码或默认密码**
4. **忽略日志中的敏感信息泄露**

## 📊 故障排查

### 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| `ECONNREFUSED` | 数据库未启动 | 检查 PostgreSQL 服务状态 |
| `password authentication failed` | 密码错误 | 验证 DB_PASSWORD 配置 |
| `role "xxx" does not exist` | 用户不存在 | 创建数据库用户 |
| `Redis connection failed` | Redis 未运行 | 启动 Redis 服务 |
| `Missing JWT secrets` | 缺少密钥配置 | 设置 JWT_SECRET 环境变量 |

### 调试命令

```bash
# 检查数据库连接
PGPASSWORD=$DB_PASSWORD psql -h $DB_HOST -p $DB_PORT -U $DB_USER -d $DB_NAME -c "SELECT 1"

# 检查 Redis 连接
redis-cli -h $REDIS_HOST -p $REDIS_PORT ping

# 检查环境变量是否加载
node -e "console.log('DB_HOST:', process.env.DB_HOST)"
```
